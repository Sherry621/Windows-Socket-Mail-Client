# 计算机网络实践项目设计文档

## 项目名称：基于 Socket 编程的 Windows 邮件客户端系统设计与实现

---

## 一、项目背景

电子邮件是互联网中最经典、最基础的应用之一。与即时通信工具相比，电子邮件具有结构清晰、协议成熟、跨平台能力强、适合正式通信等特点。在计算机网络课程中，邮件系统能够很好地体现应用层协议与传输层 TCP 连接之间的关系。

一个完整的邮件客户端通常包括邮件发送和邮件接收两部分。其中，SMTP 协议主要负责邮件发送，POP3 协议主要负责邮件接收和管理。客户端通过 TCP Socket 与邮件服务器建立连接，然后按照协议规定发送命令、接收响应、处理邮件内容。

本项目拟设计并实现一个运行于 Windows 操作系统上的图形化邮件客户端系统。系统不直接调用现成邮件库，而是基于 Socket 编程，从创建 Socket、建立 TCP 连接开始，实现 SMTP 和 POP3 协议的基本功能。用户可以通过图形化界面完成邮件编写、发送、接收、阅读和删除等操作。

---

## 二、项目目标

本项目的总体目标是实现一个基于 Socket 编程的简易邮件客户端系统，能够完成邮件发送与接收管理的基本功能。

具体目标如下：

1. 实现 Windows 平台下的图形化邮件客户端界面。
2. 使用 Socket 编程实现 SMTP 客户端功能。
3. 使用 Socket 编程实现 POP3 客户端功能。
4. 支持用户配置邮箱账号、密码、SMTP 服务器和 POP3 服务器地址。
5. 支持邮件编写功能，包括收件人、主题、正文等内容。
6. 支持通过 SMTP 协议发送邮件。
7. 支持通过 POP3 协议接收邮件列表。
8. 支持阅读邮件正文。
9. 支持删除服务器端邮件。
10. 支持邮件操作日志显示。
11. 支持基本异常处理，例如连接失败、认证失败、邮件发送失败、邮件读取失败等。
12. 支持将邮件摘要、账号配置或操作记录保存在本地数据库中。

## 三、需求分析

### 3.1 功能性需求

#### 3.1.1 用户账号配置功能

用户需要在客户端中配置自己的邮箱账户信息，包括邮箱地址、授权码或密码、SMTP 服务器地址、SMTP 端口、POP3 服务器地址和 POP3 端口。

主要功能包括：

1. 输入邮箱地址；
2. 输入邮箱授权码或密码；
3. 输入 SMTP 服务器地址；
4. 输入 SMTP 服务器端口；
5. 输入 POP3 服务器地址；
6. 输入 POP3 服务器端口；
7. 保存账号配置；
8. 加载历史账号配置。

常见服务器配置示例：

```text
SMTP 服务器：smtp.example.com
SMTP 端口：25 / 465 / 587

POP3 服务器：pop.example.com
POP3 端口：110 / 995
```

说明：如果课程要求重点是 Socket 明文协议实现，可以优先使用端口 25 和 110 进行实验；如果邮箱服务器要求 SSL/TLS，则需要额外处理安全连接。

---

#### 3.1.2 邮件编写功能

用户可以通过图形化界面编写邮件。邮件编写界面应至少包括收件人、主题和正文三个部分。

主要功能包括：

1. 输入收件人邮箱地址；
2. 输入邮件主题；
3. 输入邮件正文；
4. 支持清空当前邮件；
5. 支持发送前检查收件人和主题是否为空；
6. 支持显示发送状态。

邮件基本结构如下：

```text
From: sender@example.com
To: receiver@example.com
Subject: 邮件主题

邮件正文内容
```

---

#### 3.1.3 邮件发送功能

邮件发送功能由 SMTP 客户端模块实现。客户端通过 Socket 与 SMTP 服务器建立 TCP 连接，然后按照 SMTP 协议完成身份认证和邮件发送。

主要功能包括：

1. 建立 SMTP 服务器 TCP 连接；
2. 接收服务器欢迎响应；
3. 发送 HELO 或 EHLO 命令；
4. 执行 AUTH LOGIN 认证；
5. 发送用户名；
6. 发送密码或授权码；
7. 发送 MAIL FROM 命令；
8. 发送 RCPT TO 命令；
9. 发送 DATA 命令；
10. 发送邮件头部和正文；
11. 使用单独一行英文句点结束邮件正文；
12. 接收服务器响应，判断邮件是否发送成功；
13. 关闭 SMTP 连接。

SMTP 典型交互流程如下：

```text
S: 220 smtp.example.com Service Ready
C: HELO localhost
S: 250 OK
C: AUTH LOGIN
S: 334 VXNlcm5hbWU6
C: base64(username)
S: 334 UGFzc3dvcmQ6
C: base64(password)
S: 235 Authentication successful
C: MAIL FROM:<sender@example.com>
S: 250 OK
C: RCPT TO:<receiver@example.com>
S: 250 OK
C: DATA
S: 354 End data with <CR><LF>.<CR><LF>
C: From: sender@example.com
C: To: receiver@example.com
C: Subject: Test Mail
C:
C: Hello, this is a test email.
C: .
S: 250 OK
C: QUIT
S: 221 Bye
```

---

#### 3.1.4 邮件接收功能

邮件接收功能由 POP3 客户端模块实现。客户端通过 Socket 与 POP3 服务器建立 TCP 连接，登录成功后获取邮箱中的邮件数量和邮件列表。

主要功能包括：

1. 建立 POP3 服务器 TCP 连接；
2. 接收服务器欢迎信息；
3. 发送 USER 命令；
4. 发送 PASS 命令；
5. 获取邮箱状态；
6. 获取邮件列表；
7. 显示邮件编号、大小和摘要信息；
8. 支持刷新邮件列表。

POP3 典型交互流程如下：

```text
S: +OK POP3 server ready
C: USER sender@example.com
S: +OK
C: PASS password
S: +OK Logged in
C: STAT
S: +OK 3 20480
C: LIST
S: +OK
S: 1 1024
S: 2 4096
S: 3 15360
S: .
C: QUIT
S: +OK Bye
```

---

#### 3.1.5 邮件阅读功能

用户可以在邮件列表中选择某封邮件并阅读邮件内容。客户端通过 POP3 协议的 RETR 命令获取指定邮件的完整内容。

主要功能包括：

1. 用户选择邮件编号；
2. 客户端发送 RETR 命令；
3. 接收邮件完整内容；
4. 解析邮件头部；
5. 显示发件人、收件人、主题、日期和正文；
6. 支持返回邮件列表界面。

POP3 阅读邮件命令示例：

```text
RETR 1
```

服务器返回示例：

```text
+OK 1024 octets
From: sender@example.com
To: receiver@example.com
Subject: Hello
Date: ...

This is the mail body.
.
```

---

#### 3.1.6 邮件删除功能

用户可以选择邮件列表中的某封邮件并执行删除操作。客户端通过 POP3 协议的 DELE 命令标记该邮件为删除状态。需要注意的是，POP3 协议中邮件通常是在 QUIT 命令执行后才真正从服务器删除。

主要功能包括：

1. 用户选择需要删除的邮件；
2. 弹出确认提示；
3. 发送 DELE 命令；
4. 判断服务器是否返回 +OK；
5. 标记该邮件已删除；
6. 退出时发送 QUIT 使删除操作生效；
7. 刷新邮件列表。

POP3 删除邮件命令示例：

```text
DELE 2
```

服务器响应：

```text
+OK message 2 deleted
```

---

#### 3.1.7 日志显示功能

系统应提供日志区域，用于显示 SMTP 和 POP3 协议交互过程，方便用户观察应用层协议通信过程，也便于实验报告截图。

日志内容包括：

1. 连接服务器信息；
2. 客户端发送命令；
3. 服务器响应；
4. 登录状态；
5. 邮件发送状态；
6. 邮件接收状态；
7. 邮件删除状态；
8. 异常信息。

为了保护用户隐私，日志中不应显示明文密码或授权码，可以用星号代替。

示例：

```text
[SMTP] 正在连接 smtp.example.com:25
[SMTP-SERVER] 220 Service Ready
[SMTP-CLIENT] HELO localhost
[SMTP-SERVER] 250 OK
[SMTP-CLIENT] AUTH LOGIN
[SMTP-CLIENT] USERNAME: ****
[SMTP-CLIENT] PASSWORD: ****
[SMTP] 邮件发送成功

[POP3] 正在连接 pop.example.com:110
[POP3-SERVER] +OK POP3 server ready
[POP3-CLIENT] USER user@example.com
[POP3-CLIENT] PASS ****
[POP3] 登录成功
```

---

#### 3.1.8 本地数据库记录功能

系统可以使用本地数据库保存用户配置、邮件摘要和操作日志。数据库不是邮件协议实现的核心，但可以提高系统完整性，便于展示软件设计能力。

主要功能包括：

1. 保存邮箱账号配置；
2. 保存已接收邮件摘要；
3. 保存邮件发送记录；
4. 保存操作日志；
5. 支持重新打开程序后加载历史记录。

---

### 3.2 非功能性需求

#### 3.2.1 易用性

系统应采用图形化界面，用户不需要手动输入 SMTP 或 POP3 命令即可完成邮件发送、接收、阅读和删除操作。

#### 3.2.2 稳定性

系统需要对网络异常、服务器异常和用户输入错误进行处理，避免程序崩溃。

#### 3.2.3 可维护性

系统应采用模块化设计，将界面模块、SMTP 模块、POP3 模块、Socket 通信模块、邮件解析模块、数据库模块和日志模块分离。

#### 3.2.4 可扩展性

系统后续可以扩展附件发送、HTML 邮件解析、IMAP 协议支持、多账号管理、邮件搜索和邮件分类等功能。

#### 3.2.5 安全性

系统不应在日志或数据库中保存明文密码。如果需要保存密码，应采用加密方式。邮件传输过程中，如果使用普通 SMTP/POP3 明文端口，则用户名和密码可能以明文或简单编码方式传输，因此该系统主要用于教学实验环境。

### 5.7 Socket 通信模块设计

Socket 通信模块是 SMTP 和 POP3 协议实现的底层基础。

主要功能包括：

1. 创建 TCP Socket；
2. 连接服务器；
3. 发送字符串命令；
4. 接收服务器响应；
5. 处理多行响应；
6. 关闭 Socket；
7. 捕获网络异常。

通用方法设计：

```text
create_socket()
connect(host, port)
send_line(text)
recv_line()
recv_multiline(end_mark)
close()
```

SMTP 和 POP3 命令通常以 CRLF 结尾，即：

```text
\r\n
```

因此发送命令时应统一添加回车换行符。

### 5.8 日志与异常处理模块设计

日志模块用于记录系统运行过程。

日志类型包括：

1. INFO：普通提示；
2. SEND：客户端发送命令；
3. RECV：服务器响应；
4. ERROR：错误信息；
5. SUCCESS：操作成功。

异常处理模块需要处理以下异常：

1. Socket 连接异常；
2. 服务器无响应；
3. 认证失败；
4. 邮件发送失败；
5. 邮件接收失败；
6. 邮件解析失败；
7. 数据库读写失败；
8. 用户输入不完整。

## 十一、关键技术说明

### 11.1 Socket 编程

本项目要求不依赖高级邮件库，而是使用 Socket 方式实现 SMTP 和 POP3 协议。因此，Socket 编程是项目核心。

客户端首先创建 TCP Socket，然后连接到邮件服务器指定端口。连接建立后，客户端和服务器之间通过文本命令进行交互。

SMTP 和 POP3 均属于基于文本的应用层协议，这使得协议交互过程较为清晰，适合通过日志展示。

---

### 11.2 SMTP 协议实现

SMTP 用于发送邮件。本项目实现的 SMTP 客户端需要完成服务器连接、身份认证、指定发件人和收件人、发送邮件内容等过程。

其中 AUTH LOGIN 认证需要对用户名和密码进行 Base64 编码。需要注意的是，Base64 不是加密，只是一种编码方式。

---

### 11.3 POP3 协议实现

POP3 用于接收邮件。本项目实现的 POP3 客户端需要完成服务器连接、用户登录、邮件列表获取、邮件内容读取和邮件删除。

POP3 的响应比较简单，主要通过 +OK 和 -ERR 判断命令是否执行成功。

---

### 11.4 多行响应处理

SMTP 和 POP3 中都可能出现多行响应，尤其是 POP3 的 LIST 和 RETR 命令。

POP3 多行响应通常以单独一行英文句点结束：

```text
.
```

因此客户端在读取多行内容时，需要循环读取服务器返回内容，直到检测到单独一行 "." 为止。

---

### 11.5 邮件内容解析

邮件内容通常由邮件头和邮件正文组成，中间用空行分隔。

基本结构如下：

```text
From: sender@example.com
To: receiver@example.com
Subject: Hello
Date: Mon, 1 Jun 2026 12:00:00 +0800

This is the body of the email.
```

客户端需要将邮件头部中的 From、To、Subject、Date 等字段提取出来，并将正文显示在界面中。

---

### 11.6 字符编码处理

邮件主题和正文可能包含中文，因此系统需要考虑字符编码问题。

基本处理策略：

1. 客户端发送邮件时使用 UTF-8 编码；
2. 邮件头部声明 Content-Type；
3. 邮件正文按 UTF-8 编码发送；
4. 接收邮件时尝试解析 UTF-8、GBK 等常见编码；
5. 对无法解析的内容给出提示或显示原始内容。

发送邮件时可构造如下头部：

```text
Content-Type: text/plain; charset="utf-8"
Content-Transfer-Encoding: 8bit
```

## 十二、异常处理设计

### 12.1 SMTP 连接异常

可能原因：

1. SMTP 服务器地址错误；
2. SMTP 端口错误；
3. 网络不可达；
4. 服务器拒绝连接。

处理方式：

1. 捕获 Socket 异常；
2. 在界面提示连接失败；
3. 日志记录错误原因；
4. 恢复发送按钮状态。

---

### 12.2 SMTP 认证异常

可能原因：

1. 用户名错误；
2. 密码或授权码错误；
3. 邮箱未开启 SMTP 服务；
4. 服务器不支持 AUTH LOGIN。

处理方式：

1. 判断服务器返回码；
2. 提示认证失败；
3. 建议检查邮箱授权码；
4. 不在日志中输出明文密码。

---

### 12.3 邮件发送异常

可能原因：

1. 收件人格式错误；
2. 服务器拒绝收件人；
3. 邮件内容格式不正确；
4. DATA 阶段发送失败。

处理方式：

1. 提示发送失败；
2. 保存失败记录；
3. 日志记录服务器响应；
4. 保留邮件编辑内容，避免用户重新输入。

---

### 12.4 POP3 连接异常

可能原因：

1. POP3 服务器地址错误；
2. POP3 端口错误；
3. 网络连接失败；
4. 服务器无响应。

处理方式：

1. 捕获连接异常；
2. 提示用户检查 POP3 配置；
3. 写入错误日志。

---

### 12.5 POP3 认证异常

可能原因：

1. 用户名错误；
2. 密码或授权码错误；
3. 邮箱未开启 POP3 服务；
4. 服务器拒绝登录。

处理方式：

1. 判断服务器是否返回 -ERR；
2. 提示登录失败；
3. 日志隐藏密码内容。

---

### 12.6 邮件解析异常

可能原因：

1. 邮件格式复杂；
2. 邮件包含 HTML；
3. 邮件包含附件；
4. 邮件编码无法识别。

处理方式：

1. 优先显示原始文本内容；
2. 对无法解析字段显示“未知”；
3. 在日志中记录解析失败原因。

## 十六、预期运行效果

系统运行后，用户可以完成以下操作：

1. 打开邮件客户端系统；
2. 输入邮箱账号、授权码、SMTP 服务器和 POP3 服务器信息；
3. 保存账号配置；
4. 切换到“写邮件”页面；
5. 输入收件人、主题和正文；
6. 点击发送，系统通过 SMTP 协议发送邮件；
7. 切换到“收件箱”页面；
8. 点击刷新，系统通过 POP3 协议获取邮件列表；
9. 选择某封邮件并点击阅读；
10. 系统显示邮件发件人、收件人、主题和正文；
11. 选择某封邮件并点击删除；
12. 系统通过 POP3 协议删除服务器端邮件；
13. 用户可以在日志界面查看 SMTP 和 POP3 协议交互过程。

## 十七、项目创新点与扩展方向

### 17.1 项目创新点

1. 直接使用 Socket 实现 SMTP 和 POP3 协议，而不是调用现成邮件库；
2. 同时实现邮件发送和邮件接收功能；
3. 使用图形化界面降低用户操作难度；
4. 提供协议日志展示，便于观察应用层协议通信过程；
5. 支持邮件阅读和删除，功能较完整；
6. 使用本地数据库保存配置、邮件摘要和发送记录；
7. 模块化设计清晰，便于后续扩展。

---

## 十八、项目总结

本项目围绕计算机网络实践课程要求，设计并实现一个基于 Socket 编程的 Windows 图形化邮件客户端系统。系统以 SMTP 和 POP3 协议为核心，实现邮件编写、发送、接收、阅读和删除等基本功能。

通过本项目，小组成员能够深入理解应用层协议的工作机制，掌握 TCP Socket 编程的基本流程，熟悉客户端与服务器之间基于文本命令和响应码的通信方式。同时，项目还涉及图形化界面设计、邮件格式解析、数据库记录、异常处理和系统测试等内容，具有较强的综合实践价值。

最终系统预期形成一个功能完整、结构清晰、运行稳定的邮件客户端程序，并配套完成实验报告、系统设计说明、测试截图、小组分工说明和源码任务说明。
