# SMTP 测试清单

本文档用于成员二整理 SMTP 成功和失败测试，配合截图和实验报告使用。

## 测试前准备

开始测试前，请先确认：

```text
1. SMTP 服务器地址正确
2. SMTP 端口正确
3. 邮箱已开启 SMTP 服务
4. 使用的是授权码而不是登录密码（如服务商有要求）
5. 当前网络可访问对应 SMTP 服务器
6. 项目已成功编译
```

建议优先准备两个测试账号：

```text
发件账号：用于登录 SMTP
收件账号：用于接收测试邮件
```

## 成功测试

### T01 SMTP 连接测试

测试目标：

```text
验证能够连接 SMTP 服务器并正常退出
```

操作步骤：

```text
1. 填写邮箱、授权码、SMTP 服务器和端口
2. 点击“测试 SMTP”
3. 观察日志窗口和状态栏
```

预期结果：

```text
1. 能读取 220 欢迎响应
2. 能发送 QUIT
3. 状态栏提示连接测试完成
4. 日志中存在连接、响应和退出记录
```

建议截图内容：

```text
账号配置区域
日志区域中的 220 和 QUIT 交互
状态栏成功提示
```

### T02 普通文本邮件发送成功

测试目标：

```text
验证完整 SMTP 发送流程可用
```

操作步骤：

```text
1. 填写有效 SMTP 配置
2. 在写邮件页填写收件人、主题和正文
3. 点击“发送”
4. 观察日志、状态栏和“已发送”页
5. 到收件邮箱确认邮件是否收到
```

预期结果：

```text
1. 日志中出现 EHLO/HELO
2. 日志中出现 AUTH LOGIN
3. 日志中出现 MAIL FROM
4. 日志中出现 RCPT TO
5. 日志中出现 DATA
6. 服务端返回 250
7. 状态栏显示发送成功
8. 已发送列表新增成功记录
9. 收件邮箱可看到测试邮件
```

建议截图内容：

```text
写邮件界面发送前
日志中的完整 SMTP 交互
已发送页中的成功记录
收件邮箱收到测试邮件的界面
```

## 失败测试

### E01 SMTP 服务器地址错误

测试目标：

```text
验证连接异常处理
```

操作步骤：

```text
1. 将 SMTP 服务器改为错误地址
2. 点击“测试 SMTP”或直接发送邮件
```

预期结果：

```text
1. 连接失败
2. 状态栏提示失败
3. 日志记录错误原因
```

建议截图内容：

```text
错误配置
状态栏失败提示
日志中的异常信息
```

### E02 授权码错误

测试目标：

```text
验证 AUTH LOGIN 失败处理
```

操作步骤：

```text
1. 保持 SMTP 地址和端口正确
2. 填写错误授权码
3. 尝试发送邮件
```

预期结果：

```text
1. 能成功连接服务器
2. AUTH LOGIN 阶段失败
3. 返回认证失败信息
4. 日志中不出现明文密码或授权码
```

建议截图内容：

```text
日志中的 AUTH LOGIN 阶段
失败提示信息
敏感信息已脱敏的日志内容
```

### E03 收件人地址错误或被服务器拒绝

测试目标：

```text
验证 RCPT TO 或发送结果失败处理
```

操作步骤：

```text
1. 使用正确的 SMTP 配置
2. 输入错误收件人地址
3. 发送邮件
```

预期结果：

```text
1. AUTH LOGIN 成功
2. MAIL FROM 成功
3. RCPT TO 被拒绝，或后续发送失败
4. 返回明确错误信息
```

建议截图内容：

```text
日志中的 RCPT TO
服务端拒绝响应码
状态栏失败提示
```

### E04 端口错误或网络不可达

测试目标：

```text
验证网络层失败时的错误处理
```

操作步骤：

```text
1. 将 SMTP 端口改为错误值
2. 或在测试中断开网络
3. 再次尝试连接或发送
```

预期结果：

```text
1. 无法建立 TCP 连接
2. 程序不崩溃
3. 日志记录错误信息
```

建议截图内容：

```text
错误端口配置
日志中的连接失败信息
```

## 日志检查点

截图前建议确认日志中包含以下关键节点：

```text
220
EHLO 或 HELO
AUTH LOGIN
MAIL FROM
RCPT TO
DATA
250 发送成功
221 QUIT
```

失败场景则重点保留：

```text
连接失败原因
认证失败响应
收件人拒绝响应
敏感信息已脱敏
```
